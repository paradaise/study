Настройка зонального межсетевого экрана (ZPF)

Суть задания:
Настроить на граничном маршрутизаторе R3 односторонний межсетевой экран нового поколения (ZPF), который:

- Разрешает трафик из внутренней сети (192.168.3.0/24) во внешнюю.

- Блокирует весь трафик из внешней сети во внутреннюю.

Это классическая политика "изнутри — наружу можно, снаружи — внутрь нельзя".




По ТЗ реализовал 2 политики IN-ZONE и OUT-ZONE и применили их на интерфейс

0) Сначала проверил что абсолютно весь трафик ходит между узлами сети.

1) Проверка политики IN-ZONE->OUT-ZONE ( разрешенный трафик )
Проверим, что внутренние хосты не потеряли доступ к внешним ресурсам и ZPF видит сесансы

1. ping 192.168.1.3 с PC-C ( успешно )
2. ssh -l Admin 10.2.2.2 с PC-C ( успешно )
3. PC-C->Desktop->Web Browser. URI: http://192.168.1.3
4. ciscoconpa55
en ( пароль  ciscoenpa55)
   #show policy-map type inspect zone-pair sessions

2) Проверка полиик OUT-ZONE->IN-ZONE ( заблокированный трафик )
Проверим, что обратный трафик, который был инициирован извне, блокируется.

1.ping 192.168.3.3 с PC-A ( неуспешен)
2.ping 192.168.3.3 с R2 ( неуспешно )
