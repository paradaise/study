Суть работы:
Настройка межсетевого экрана Cisco ASA как граничного устройства безопасности, разделяющего сеть на зоны (Inside, Outside, DMZ) и обеспечивающего безопасный доступ.

Что сделали и как работает:
1. Базовая настройка ASA
Настроили зоны безопасности:

Inside (VLAN 1) - уровень 100 - доверенная сеть
Outside (VLAN 2) - уровень 0 - ненадежная сеть (интернет)
DMZ (VLAN 3) - уровень 70 - промежуточная зона

Принцип: Трафик может течь из зоны с высоким уровнем безопасности в зону с низким, но не наоборот (без правил)

2. Преобразование адресов (NAT/PAT)
PAT для внутренней сети: Все хосты Inside (192.168.1.0/24) выходят в интернет через один внешний IP

Static NAT для DMZ: Сервер в DMZ (192.168.2.3) доступен извне по публичному IP 209.165.200.227

3. Политики безопасности
ACL: Разрешили доступ из интернета только к веб-серверу в DMZ (порт 80)
MPF: Настроили инспектирование ICMP - разрешили ответы на ping

4. Сервисы
DHCP: Раздаем IP-адреса внутренним хостам
SSH: Настроили безопасный удаленный доступ для администраторов
AAA: Локальная аутентификация пользователей

Основные команды для проверки:
1. Проверка интерфейсов и зон
CCNAS-ASA# show interface ip brief
CCNAS-ASA# show ip address
CCNAS-ASA# show switch vlan

Видим, что  все три логических интерфейса созданы, назначены правильные IP-адреса и имена зон
Физические порты правильно распределены по VLANам (зонам безопасности)

2. Проверка маршрутизации
CCNAS-ASA# show route

Настроен маршрут по умолчанию через шлюз 209.165.200.225 в интернет

3. Проверка NAT
CCNAS-ASA# show run object
CCNAS-ASA# show nat

PAT для внутренней сети через внешний интерфейс
Static NAT для сервера в DMZ на публичный IP 209.165.200.227

4. Проверка политик доступа
CCNAS-ASA# show access-list
CCNAS-ASA# show service-policy

ACL разрешает доступ из интернета только к веб-серверу и ICMP в DMZ

5. Проверка DHCP
CCNAS-ASA# show dhcpd state
CCNAS-ASA# show dhcpd binding

DHCP сервер включен на внутреннем интерфейсе

6. Проверка SSH
CCNAS-ASA# show ssh sessions
Что показать: Активные SSH-сессии


Итог: Создана безопасная сетевая периметра с контролем доступа, скрытием внутренней структуры и изоляцией сервисов.


Inside (VLAN 1) - уровень 100 - доверенная сеть
- PC-B - 192.168.1.3
- Внутренние пользователи компании

Outside (VLAN 2) - уровень 0 - ненадежная сеть (интернет)
- R1 (G0/0) - 209.165.200.225
- R2 - через R1
- PC-C - 172.16.3.3 (удаленный администратор)
- Интернет-хосты

DMZ (VLAN 3) - уровень 70 - промежуточная зона
- Сервер DMZ - 192.168.2.3 (веб-сервер)
- Публичные сервисы компании

Как это работает на практике:
PC-B (Inside) → Интернет: Разрешено (из высокой безопасности в низкую).PAT подменяет его IP на внешний
Интернет → PC-B: Запрещено (из низкой безопасности в высокую).Static NAT преобразует публичный IP в приватный сервера + ACL проверяет разрешение
Интернет → Сервер DMZ: Разрешено только по правилам ACL (веб-трафик)
PC-B → Сервер DMZ: Ограничено (из уровня 100 в уровень 70)

Администратор управляет ASA через SSH безопасно
Внутренние устройства получают IP автоматически от DHCP
DMZ специально имеет уровень 70 - чтобы интернет мог обращаться к серверу (через ACL), но внутренние пользователи не могли случайно попасть на него напрямую без правил.
